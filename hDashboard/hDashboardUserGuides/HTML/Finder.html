Documents on the website are managed using a web-based utility called Finder. Finder is inspired by and based on the document management utility found in Mac OS X of the same name.

To launch Finder, simply navigate to the Dashboard and click on “Documents”.

Note: You must have administrative access to launch Finder.

Finder launches with a view that might look similar to the file system of your local computer. To keep things simple, some advanced features displayed in this screenshot may be turned off.

Files can be uploaded to any folder that you see here.

Browsers Support of Advanced Features

While Hot Toddy is designed to support as much as possible in every desktop browser, some features are so new to web development that they are not yet supported in every desktop browser and in particular by a plain vanilla install of Microsoft’s Internet Explorer browser.

What isn’t supported?

The most useful features, of course!

  • The ability to drag and drop files from your desktop into the web-based Finder
  • The ability to select multiple files for upload using the “Upload File” button.
  • Editing documents on the website using the “Edit This Page” link.

I’ll get to explaining the various ways that you can upload files in the next section, but it’s important to understand why all features cannot be supported by Internet Explorer in general.

Hot Toddy implements some cutting edge technology that has only recently been adopted by some 3rd-party browsers. One of those features is the ability to drag and drop multiple documents into the browser window and upload those files to the website seamlessly.

The browsers supporting these features are as follows: Apple Safari, Google Chrome, Mozilla Firefox, or Microsoft’s Internet Explorer with the Google Chrome Frame plugin installed.

Uploading Documents

Documents can be uploaded to the website using one of the following options:

  • Upload a single file at a time by clicking the “Upload File” button, or the “Upload File” option from one of the context menus. This option is supported in all browsers.
  • Upload multiple files at once by drag and drop from your desktop using a supported browser (see previous section about Browser Support).
  • Select multiple files for upload by clicking the “Upload File” button as you would do when uploading a single file at a time, but select multiple files while browsing your local file system by holding down the “Control” or “Shift” keys on your keyboard while selecting files. This too only works with a supported browser (see previous section about Browser Support).

Uploading a Single File at a Time

To upload a single file, click the “Upload File” button at the bottom of the Finder window.

Uploading Multiple Files by Drag and Drop

To upload multiple files by drag and drop, select one or more files from your desktop or a folder.

Note: The items that you select for upload may only be files, drag and drop of folders is not supported.

In the preceding screenshot, you see a file dragged from the Mac OS X desktop into the Safari browser window.

To accomplish the same with Internet Explorer on a Windows PC, you must install the Google Chrome Frame plugin, which will enable drag and drop from the desktop to the browser window. Alternatively, you can install a browser other than Internet Explorer, such as Safari, Google Chrome or Firefox

After dropping the file onto the browser window you should see an upload queue appear, letting you know that upload activity is taking place.

Note: While drag and drop is supported for uploading documents to the website using a supported browser, this feature does not work in reverse, i.e., drag and drop of files or folders from the Finder window to the desktop. Unfortunately, no browser yet supports a way to do this.

Uploading Multiple Files by Selecting Multiple Files

The last option for uploading multiple files allows you to select multiple files when clicking on the “Upload File” button.

When you click “Choose File”, instead of selecting just one file to upload, you have the option of selecting multiple files by holding down the “Control” or “Shift” keys on your keyboard to select multiple files for upload. In the preceding screenshot, I selected four files to upload. Again, this feature only works with a supported browser.

Note: Like drag and drop from the desktop, this feature only supports selection of one or more files, and unfortunately, selecting a folder for upload is not supported.

Caveats Regarding File Uploads

There are some hard limits regarding the quantity of files that you may upload at once, and the collective size of the files uploaded.

  • The number of files uploaded cannot exceed 500
  • The size of all files uploaded cannot exceed 500MB

The preceding numbers are theoretical limitations, and you will likely run into difficulty as you come closer to these limits.

Extremely large files can be added to the website, but should be added by other means.

Since large files cannot be added using the web-based tool, adding large files requires the assistance of your website's developers or administrators.

Additionally, since you are transmitting files from your location, across the internet to the web server, bigger file uploads may take a lot of time to complete depending on the size of the files involved and the speed of your connection to the internet.

Organizing Files and Folders

Finder supports drag and drop within the window itself. Files and folders existing on the website can be moved and reorganized anyway you like by drag and drop. Using a supported browser, you can even open multiple Finder windows and drag and drop between them.

Renaming Files, Folders and Categories

Renaming a file, folder or category can be done one of two ways:

  1. Double-Click on the name of the file, folder or category in the right-side view and edit the name.
  2. Select a file, folder, or category in the right-side view and press Return or Enter on your keyboard, then edit the name.

Hidden Files, Folders and Categories

Finder uses a special naming convention to hide files, folders, and categories from view. This feature is used to implement technical features within the website that regular users should never see or interact with.

Any file, folder, or category named with a leading period character will be hidden from you and become inaccessible. For example “.hidden_file.pdf”

For this reason, please avoid naming files, folders or categories with leading period characters.

Keyboard Shortcuts

Finder implements a few useful keyboard shortcuts:

  • Control + R / Command + R = Refresh
  • Directional Arrows / Tab (after selecting a file, folder or category) = Change the selected file, folder, or category.
  • Control + Shift + Up Arrow / Command + Shift + Up Arrow = Move up a folder
  • Control + O / Command + O (with a folder selected) = Open a folder
  • Delete / Backspace = Delete the selected file, folder or category.
  • Control + [ = Go Back
  • Control + ] = Go Forward
  • Control + Shift + A / Command + Shift + A = Open the Applications Folder
  • Return / Enter = Rename the selected file, folder, or category

Using Categories

Beyond uploading and organizing your documents into folders, the website provides an additional option for organizing files called Categories.

Categories are distinguished from normal folders in the following ways:

  1. A file can appear in only one folder, but can appear in as many Categories as you like.
  2. Categories have green folder icons.

What is a category and why is it useful?

Categories provide you with a way to both organize and tag documents. For example, you may upload a file and consider it relevant to “Doctors”, “Nurses”, and “Practices”. Without categorization, the only possible option may be to upload the same document three times, to three separate folders for “Doctors”, “Nurses” and “Practices”. With categories, you upload the document just once to one folder, and then tag the file with the categories you deem relevant.

In fact, Finder actually keeps a record of every file uploaded and prevents you from uploading the same file more than once. If you attempt to upload the same document multiple times, Finder will present an error letting you know that the document already exists on the website in addition to where the file presently resides.

Instead of uploading a document multiple times, you have the option to tag a file with as many categories as you like. Categories are simply additional meta data added to the file, making a file easier to find and associate with different contexts.

To categorize a document, you must first upload a document to a regular folder (blue folder icon). This places the document in the website’s file system.

Once uploaded, you can drag and drop any file to a green folder to add it to that category.

To remove a file from a category, right-click on the file and click “Remove From Category”.

Once added to a category, the file becomes available using the various category browsing and searching options available on the intranet website.

Updating Files

When you add a file to the website, instantly heaps of data about that file begin to be collected. Who viewed the file. The last time a person accessed that file. Categories that file is added to. Permissions applied to the file. And most importantly, people will be sharing and bookmarking URLs to files.

This enables features like searching your most viewed and most accessed documents.

For this reason, when you update a file, it is a very bad idea to delete a file and upload its replacement. When you update a file in this way, all of that extra information is lost and you have to start over with a blank slate.

Luckily, Finder supports a better way to update a file, which is done by clicking the “Edit File” button at the bottom of the Finder window.

To use this feature, first select a file by clicking on it in the right-side view, then click “Edit File”.

To update the file itself, select a replacement file, then click “Save”. The new file will be uploaded and will replace the content of the old file, but all of that extra information will remain in place.
